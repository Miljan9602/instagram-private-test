If the first letter of a pattern is "/", we treat it as a regex. The exceptions should be roughly arranged by how common they are, with the most common ones checked first, at the top. Note that not all exceptions are listed below. Some are thrown via other methods than this automatic message parser. WARNING TO CONTRIBUTORS: Do NOT "contribute" a bunch of endpoint function specific garbage exceptions here, such as "User not found", "Duplicate comment", "No permission to view profile" or other garbage. Those server messages are human-readable, unreliable (they can change at any time) and are also totally non-critical. You should handle them yourself in your end-user applications by simply catching their generic EndpointException and looking at the contents of its getMessage() property, or checking if it hasResponse() and then getResponse() to see the full server response. The exceptions listed below are *critical* exceptions related to the CORE of the API! Nothing else.

public static autoThrow( $prefixString : string|null , $serverMessage : string|null [, $serverResponse : Response|null = null ] [, $httpResponse : ResponseInterface|null = null ] ) : mixed

Uses the generic EndpointException if no other exceptions match.

Parameters
$prefixString : string|null

What prefix to use for the message in the final exception. Should be something helpful such as the name of the class or function which threw. Can be NULL.

$serverMessage : string|null

The failure string from Instagram's API (from getMessage()). Might be empty in some cases.

$serverResponse : Response|null = null

The complete server response object, if one is available (optional).

$httpResponse : ResponseInterface|null = null

The HTTP response object (if available).

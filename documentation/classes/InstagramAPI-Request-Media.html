getComments()

Media.php : 426

Get media comments.

public getComments( $mediaId : string [, $options : array = [] ] ) : MediaCommentsResponse

Note that this endpoint supports both backwards and forwards pagination. The only one you should really care about is "max_id" for backwards ("load older comments") pagination in normal cases. By default, if no parameter is provided, Instagram gives you the latest page of comments and then paginates backwards via the "max_id" parameter (and the correct value for it is the "next_max_id" in the response).

However, if you come to the comments "from a Push notification" (uses the "target_comment_id" parameter), then the response will ALSO contain a "next_min_id" value. In that case, you can get newer comments (than the target comment) by using THAT value and the "min_id" parameter instead.

Parameters
$mediaId : string

The media ID in Instagram's internal format (ie "3482384834_43294").

$options : array = []

An associative array of optional parameters, including: "max_id" - next "maximum ID" (get older comments, before this ID), used for backwards pagination; "min_id" - next "minimum ID" (get newer comments, after this ID), used for forwards pagination; "target_comment_id" - used by comment Push notifications to retrieve the page with the specific comment. "inventory_source" - 'media_or_ad' or 'follow_hashtag_story'.

getCommentReplies()

Media.php : 486

Get the replies to a specific media comment.

public getCommentReplies( $mediaId : string , $commentId : string [, $options : array = [] ] ) : MediaCommentRepliesResponse

You should be sure that the comment actually HAS more replies before calling this endpoint! In that case, the comment itself will have a non-zero "child comment count" value, as well as some "preview comments".

If the number of preview comments doesn't match the full "child comments" count, then you are ready to call this endpoint to retrieve the rest of them. Do NOT call it frivolously for comments that have no child comments or where you already have all of them via the child comment previews!

Parameters
$mediaId : string

The media ID in Instagram's internal format (ie "3482384834_43294").

$commentId : string

The parent comment's ID.

$options : array = []

An associative array of optional parameters, including: "max_id" - next "maximum ID" (get older comments, before this ID), used for backwards pagination; "min_id" - next "minimum ID" (get newer comments, after this ID), used for forwards pagination.
